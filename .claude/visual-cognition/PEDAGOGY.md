# PEDAGOGY.md — 教学理论 × 视觉表达指南

> 此文件在 Step 2（叙事骨架）和 Step 4（Visual Translation）时读取。
> 理论用于 AI 判断，不向用户展示行话。用户问"为什么"时用日常语言解释。

---

## 一、叙事结构（Step 2 使用）

根据内容类型和受众目标，从以下结构中选一个，说明推荐原因：

| 结构 | 最适合 | 核心逻辑 | 钩子句公式 |
|------|--------|----------|------------|
| **反差型** | 纠正误解，建立新认知 | 你以为→其实 | "你以为 X，其实 Y" |
| **问题-解答型** | 教一个概念/方法 | 为什么→因为→所以 | "为什么 X？答案出乎意料" |
| **递进/解包型** | 复杂概念层层深入 | 表面→本质→延伸 | "X 背后藏着一件更重要的事" |
| **身份认同型** | 激励、赋能、改变认知 | 你也有这个→这是你的优势 | "你比你以为的更适合做 X" |
| **故事型** | 经历、案例、产品 | 情境→冲突→转折→启示 | "那天发生了一件奇怪的事" |
| **框架型** | 传授方法论 | N个要素→每个→怎么用 | "把 X 拆开来，只有 N 件事" |
| **对比型** | 两个选项/时代/观点 | A vs B 的结构化比较 | "A 和 B，你可能选错了" |
| **时间线型** | 历史、演化、预测 | 过去→现在→未来 | "X 年前没人想到 Y 会发生" |

## 二、知识类型诊断（Step 4 使用）

每个要讲的内容块，先判断知识类型，再选表达方式：

### 2.1 概念性知识（"X 是什么"）
理解标准：学习者能用自己的话解释，能识别正确/错误的例子。

**最佳表达方式（按效果排序）：**
1. **类比动画**：把抽象概念映射到已知事物，用动画演示对应关系
2. **定义拆解**：关键词逐个高亮+解释，其余文字淡出
3. **正例/反例对比**：两栏同时显示，动画标注差异点
4. **边界测试**：展示"边缘案例"，帮助精确理解概念范围

**禁止**：直接贴维基百科式定义，没有具体例子。

---

### 2.2 程序性知识（"怎么做 X"）
理解标准：学习者能独立完成操作，不需要提示。

**最佳表达方式：**
1. **步骤动画**：每步依次出现，配进度指示器，前一步保持可见
2. **分支流程图**：遇到判断点时，展示"如果A→这样；如果B→那样"
3. **错误演示→纠正**：先展示常见错误做法，再展示正确做法（强化记忆）
4. **速记卡**：把步骤压缩成最短可记的口诀或关键词序列

**禁止**：把5个步骤放在一张 slide，没有顺序感。

---

### 2.3 叙事性知识（故事、案例、经历）
理解标准：学习者能回忆情节，能提取其中的规律或教训。

**最佳表达方式：**
1. **场景构建**：用视觉元素建立情境（人物/地点/时间），再展开事件
2. **时间轴动画**：事件按时间顺序出现，节点可点击展开细节
3. **情绪曲线**：用线形动画展示事件过程中的情绪/状态变化
4. **"那一刻"定格**：关键转折点单独一张，视觉上留白+大字

**禁止**：把故事压缩成3个 bullet point。

---

### 2.4 关系性知识（"X 和 Y 的关系"）
理解标准：学习者能描述关系的方向、强度、条件。

**最佳表达方式：**
1. **动态连接图**：节点先出现，再用动画画出连接线，线的粗细=强度
2. **维恩图动画**：两个圆各自出现，再相互靠近展示交集
3. **矩阵展开**：2×2矩阵，每个象限依次出现，最后高亮关键象限
4. **因果链动画**：A→B→C 的多米诺式连锁反应动画

**禁止**：静态箭头图，没有展示"为什么"有这个连接。

---

### 2.5 元认知知识（"为什么你要这样想"）
理解标准：学习者能监控和调整自己的思维方式。

**最佳表达方式：**
1. **两种心态对比动画**：左边"旧认知"运作方式，右边"新认知"运作方式
2. **思维过程可视化**：把内心的思考步骤外化显示（像漫画思维泡泡）
3. **误区识别练习**：展示一个情境，让观众先猜，再揭示正确思路
4. **自我提问序列**：一系列问题依次浮现，引导反思

**禁止**：说教式的"你应该这样想"，没有让观众参与判断。

---

### 2.6 数据性知识（数字、趋势、规模感）
理解标准：学习者能理解数字的含义，能比较大小和变化。

**最佳表达方式：**
1. **计数动画**：数字从0增长到目标值，配合视觉放大
2. **比例可视化**：用面积/高度/数量的视觉比例代替纯数字
3. **对比基准**：把大数字换算成日常参照物（"相当于X个XX"）
4. **趋势线动画**：折线图的线条用绘制动画逐步画出
5. **分布动画**：点/条 从无序状态聚集成有序分布

**禁止**：直接贴数字表格，没有视觉化处理。

## 三、解释方式 × 动画选择（Step 4 核心决策）

每个"认知动作"对应推荐的动画表达方式：

### 3.1 建立类比
**目标**：用已知激活先验知识，为新概念建造桥梁

动画设计模式：
```
① 已知概念出现（左侧）
② 新概念出现（右侧，初始半透明）
③ 动画线条连接两者的对应部分
④ "="或"≈"符号出现
⑤ 新概念完全显示
```

优质类比标准：
- 结构同构（类比的运作方式 = 概念的运作方式）
- 受众日常可见（不用专业知识理解类比本身）
- 一句话说完（"X 就像 Y，Z"）
- 能暴露概念的边界（类比在哪里失效，就是概念的边界）

典型例子：
- API = 餐厅点菜（你下单，厨房执行，你不需要知道怎么做菜）
- 认知负荷 = 手机内存（满了就卡顿）
- MCP = USB-C（一个接口，连所有设备）
- 神经网络 = 大脑突触（用得越多越强化）

---

### 3.2 展示过程/机制
**目标**：让看不见的工作原理变得可见

动画设计模式：
```
① 起点状态静止显示
② 触发事件（用户点击或自动）
③ 中间过程以动画展示（物质流动、状态转变、信号传递）
④ 结果状态显示
⑤ 可重复播放
```

适用场景：
- 物理/化学反应过程
- 算法执行过程（冒泡排序、二分查找）
- 生物机制（光合作用、神经信号传导）
- 社会现象动态（经济泡沫、传染病扩散）
- AI/计算机工作原理

---

### 3.3 对比差异
**目标**：通过对比让差异变得显著，纠正先验误解

动画设计模式：
```
① 两个选项同时出现（或先A后B）
② 关键差异点高亮标注（其余元素淡出）
③ 差异后果/影响以动画展示
④ 结论/推荐显示
```

对比类型：
- **Before/After**：改变前后
- **错误/正确**：先演示错误做法（强化印象），再演示正确做法
- **旧范式/新范式**：两个时代的思维方式
- **表面/本质**：第一印象 vs 深层逻辑

---

### 3.4 展示规模/比例
**目标**：让抽象的数量/规模产生直觉感受

动画设计模式：
```
① 参照物出现（已知大小）
② 目标对象以正确比例出现
③ 或：用数量动画（点/方块）填充屏幕表示比例
④ 数字标注
```

规模感工具箱：
- 太阳系比例（展示相对大小）
- 时间轴（1秒=1年）
- 像素网格（100个格子，填满X个）
- 堆叠（把N个物体叠在一起）

---

### 3.5 讲历史/演化
**目标**：展示事物随时间的变化，建立因果链

动画设计模式：
```
① 时间轴从左向右生长
② 每个关键节点停顿，显示说明
③ 可以用视觉元素的变形表示演化（形状/颜色/大小变化）
④ 标注"为什么在这里发生变化"
```

历史类型：
- 技术演化（版本迭代）
- 思想演化（学派发展）
- 社会变迁（人口/城市/制度）
- 个人成长（能力/身份变化）

---

### 3.6 建立框架/模型
**目标**：给复杂现实提供一个可操作的简化结构

动画设计模式：
```
① 复杂混乱的状态（随机分布的元素）
② 分类/整理动画（元素移动归位）
③ 框架结构显示（矩阵/层级/圆圈）
④ 每个格子/层次的意义单独显示
⑤ 举例填入
```

常用框架形式：
- 2×2 矩阵（坐标轴）
- 同心圆（层次/优先级）
- 金字塔（层级）
- 流程循环（PDCA/OODA）
- 光谱/连续轴（程度）

## 四、认知科学原则速查（AI 判断依据）

| 原则 | 核心意思 | 在 slide 里的操作 |
|------|----------|-------------------|
| **认知负荷理论** | 工作记忆容量有限 | 一张slide一个认知单元；删掉装饰性元素 |
| **双通道理论** | 视觉+语言双通道同时激活 | 动画+口播讲不同信息，不重复 |
| **先验知识激活** | 新知识挂在已有图式上 | 每个新概念找一个日常类比 |
| **逆向设计** | 先定终点再设计路径 | 先问"听完要做/感受什么"再决定内容 |
| **叙事传输** | 进入叙事时批判性降低 | 故事/案例优先于抽象定义 |
| **好奇缺口** | 信息差激发求知欲 | 钩子句制造问题，不立即给答案 |
| **间隔效应** | 分散练习比集中好 | 微课末尾加 quiz，讲座中加停顿点 |
| **检索练习** | 主动回忆比被动看好 | 让观众先猜，再揭示答案 |
| **具身认知** | 身体经验影响抽象理解 | 用运动/力量/温度类比抽象概念 |
| **情绪记忆** | 有情绪激活的内容记得更牢 | 用真实故事/意外数据制造情绪时刻 |

## 六、不同受众的解释策略

| 受众 | 先验知识假设 | 解释策略 | 避免 |
|------|------------|----------|------|
| **K12学生** | 日常生活经验 | 具体故事+游戏化类比 | 专业术语、纯数字 |
| **大学生** | 学科基础概念 | 框架+案例+批判性提问 | 说教、过于简化 |
| **教师/研究者** | 领域专业知识 | 跨领域类比+反直觉数据 | 低估他们、过于详细 |
| **企业听众** | 行业经验 | 问题→解决方案→ROI | 学术风格、过多理论 |
| **社媒观众** | 兴趣相关 | 情绪钩子→快速价值+行动 | 没有情绪激活点 |
| **技术人员** | 系统性思维 | 精确定义+边界案例 | 过于简化、类比不严谨 |
