# GUARDRAILS.md — 质量守卫系统

> `/gen-video` 流程中 Step 6 使用。确保视频内容忠实、逻辑清晰、平台可用。

---

## 一、红线（不可违反）

以下规则为硬性约束，任何自检轮次均不可放行：

1. **事实忠实**：每个事实断言（数据、案例、因果关系）必须在 `source.md` 中有据可查
2. **禁止编造**：不得凭空创造数据点、实验结果、案例故事
3. **禁止篡改因果**：不得扭曲 `source.md` 中的因果逻辑（如把「相关」说成「因果」）
4. **语气一致**：保持 `script-full.md` 的口播语气风格，不在竖版中突变为另一种人格
5. **来源可溯**：若引用具体数字/实验，必须能在 `source.md` 中指出出处段落

---

## 二、自检维度（4 项）

每轮自检逐项评估，标记 ✅ / ⚠️ / ❌：

### 2.1 知识准确性

| 检查项 | 方法 |
|--------|------|
| 数据点一致 | 逐个核对 vertical.html 中出现的数字与 source.md 原文 |
| 因果未扭曲 | 检查每个「因为…所以…」是否与 source.md 逻辑链条一致 |
| 类比未失真 | 检查类比/比喻是否保留了原始关系，未过度简化 |
| 未遗漏关键限定 | 原文中的「大多数」「某些情况下」等限定词是否保留 |

### 2.2 逻辑清晰度

| 检查项 | 方法 |
|--------|------|
| Slide 顺序合理 | 叙事结构连贯，无跳跃或重复 |
| {step} 认知有意义 | 每个 step 推进一个认知单元，非机械切分 |
| 前后衔接 | 相邻 Slide 之间有逻辑承接（非孤立话题） |
| 信息递进 | 复杂度或情感强度有明确走向 |

### 2.3 平台适配性

| 检查项 | 方法 |
|--------|------|
| 安全区域 | 所有内容在 Y:80~1340px, X:60~960px 范围内 |
| 字幕无遮挡 | 字幕区（Y:1520~1760px）无其他内容覆盖 |
| 时长合理 | 总时长 ≤ 3 分钟（抖音优选 ≤ 1 分钟） |
| 前 3 秒钩子 | 封面 Slide 有视觉冲击 + 悬念/反常识 |
| 右侧避让 | 关键信息不在右侧 120px 内 |

### 2.4 执行可行性

| 检查项 | 方法 |
|--------|------|
| HTML 可渲染 | vertical.html 语法正确，无未闭合标签 |
| GSAP 引用正确 | `../vendor/gsap.min.js` 路径存在 |
| TTS 文本无异常 | script-full.md 无乱码、无过长单段（>200字） |
| ffmpeg 参数正确 | 分辨率 1080×1920、编码 libx264、音频 aac |
| 文件完整 | narration.mp3 + timing.json + subtitles.srt 均存在 |

## 三、自检循环流程

```
┌─────────────────────────────┐
│  生成/修改产物               │
└──────────┬──────────────────┘
           ▼
┌─────────────────────────────┐
│  自检 4 维度                 │
│  ┌─ 2.1 知识准确性          │
│  ├─ 2.2 逻辑清晰度          │
│  ├─ 2.3 平台适配性          │
│  └─ 2.4 执行可行性          │
└──────────┬──────────────────┘
           ▼
      全部 ✅ ？ ──是──→ 通过，进入下一步
           │
          否（有 ⚠️/❌）
           │
           ▼
      轮次 < 3 ？ ──是──→ 修复问题，回到自检
           │
          否（已 3 轮）
           │
           ▼
┌─────────────────────────────┐
│  标记未解决问题              │
│  输出问题清单                │
│  交用户决策                  │
└─────────────────────────────┘
```

**最多 3 轮**。每轮只修复本轮发现的问题，不引入新改动。

## 五、红线违反处理

若任何轮次发现红线违反：

1. **立即停止**当前流程
2. **明确标注**哪条红线被违反
3. **给出修复建议**
4. **等待用户确认**后才继续

红线违反不计入 3 轮上限——修复红线后重新开始计轮。

## 六、降级策略

当特定组件不可用时，允许降级但需明确告知：

| 组件不可用 | 降级方案 | 影响 |
|-----------|---------|------|
| edge-tts（Python 环境缺失） | 使用 WPM 估算时长，无音频轨 | 视频无声，需手动配音 |
| ffprobe（时长获取失败） | 按文件大小估算 | 时间戳可能有 ±10% 偏差 |
| 字幕字体缺失 | 使用系统默认字体 | 字幕显示可能不美观 |
| vertical.html 渲染异常 | 保留截帧供人工检查 | 需手动排查 HTML 问题 |

降级时在自检报告中标注：
```
⚠️ 降级模式：[组件名] 不可用，已使用 [降级方案]
```
